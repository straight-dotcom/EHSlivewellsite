# EHS LiveWell — Wellness Resource Platform

A complete wellness resource hub for your organization, hosted on GitHub Pages with a Google Sheets approval backend.

## Architecture

```
Google Form  →  Google Sheet  →  Admin Email (Approve/Reject buttons)
                                        ↓
                              Apps Script processes action
                                        ↓
                        Google Calendar (events) + GitHub Actions webhook
                                        ↓
                        GitHub Actions fetches sheet → generates JSON
                                        ↓
                            GitHub Pages serves static site
                                        ↓
                         Site loads JSON and renders content
```

## Setup Guide

### Step 1: Create the Google Form

1. Go to [Google Forms](https://forms.google.com) and create a new form
2. Add these questions:
   - **Resource Type** (dropdown): Event, Recommendation, Provider, Wellness Resource
   - **Title** (short answer, required)
   - **Description** (paragraph)
   - **Link** (short answer)
   - **File Upload** (allow PDFs, images)
   - **Start Date/Time** (date-time) — use section logic to show only when Type = Event
   - **End Date/Time** (date-time) — show only when Type = Event
   - **Calendar Category** (dropdown): Physical, Mental, Nutrition, Social, Professional — show only when Type = Event
3. In Form Settings:
   - Turn on "Collect email addresses"
   - Optionally add "Your Name" field
4. Link responses to a new Google Sheet (Responses tab → green Sheets icon)

### Step 2: Set Up the Google Sheet

1. Open the linked Sheet
2. Rename the response tab to **Submissions**
3. Add columns after the last form column:
   - **Status** — add data validation: Pending, Approved, Rejected
   - **Approved By**
   - **Approval Timestamp**
   - **Drive File Links**
4. Verify column positions match `CONFIG.COLUMNS` in `apps-script/Code.gs`

### Step 3: Install the Apps Script

1. In Google Sheets: **Extensions → Apps Script**
2. Replace default code with contents of `apps-script/Code.gs`
3. Create new script file `approval` with contents of `apps-script/approval.gs`
4. Update `CONFIG` in Code.gs:
   - `ADMIN_EMAILS` — your admin email addresses
   - `CALENDAR_IDS` — Google Calendar IDs for each category
   - `GITHUB_REPO` — your GitHub username/repo
5. See `apps-script/README.md` for detailed instructions

### Step 4: Deploy Apps Script as Web App

1. Click **Deploy → New deployment**
2. Type: **Web app**
3. Execute as: **Me** / Who has access: **Anyone**
4. Copy the web app URL
5. Paste into `WEB_APP_URL` in approval.gs
6. Set `APPROVAL_SECRET` to a random string
7. Re-deploy with new version

### Step 5: Set Up Triggers

In Apps Script editor → **Triggers**:

| Function | Event Source | Event Type |
|----------|-------------|------------|
| `onFormSubmit` | From spreadsheet | On form submit |
| `onSheetEdit` | From spreadsheet | On edit |

### Step 6: Create Google Calendars

Create 5 Google Calendars:
- Physical Wellness
- Mental Wellness
- Nutrition
- Social Wellness
- Professional Development

Copy each Calendar ID from **Settings → Integrate calendar** into `CONFIG.CALENDAR_IDS`.

### Step 7: Set Up GitHub Repository

1. Create a new GitHub repo (e.g., `ehslivewell`)
2. Push this code to the repo
3. Go to **Settings → Pages**:
   - Source: Deploy from a branch
   - Branch: `main` / root
4. Add repository secrets (**Settings → Secrets → Actions**):
   - `SHEET_CSV_URL` — the "Publish to web" CSV URL for your Sheet
     - In Google Sheets: **File → Share → Publish to web**
     - Select the Submissions tab → CSV format → Publish
     - Copy the URL

5. Also add `GITHUB_TOKEN` as a Script Property in Apps Script (see Step 3)
   - Create a GitHub Personal Access Token with `repo` scope
   - Store it in Apps Script: **Project Settings → Script Properties**

### Step 8: Custom Domain (ehslivewell.com)

1. In your domain registrar, add these DNS records:
   - **A records** pointing to GitHub Pages IPs:
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```
   - **CNAME** for `www`: `your-username.github.io`
2. In GitHub repo **Settings → Pages → Custom domain**: enter `ehslivewell.com`
3. Check "Enforce HTTPS"
4. Create a `CNAME` file in the repo root containing: `ehslivewell.com`

### Step 9: Update the Submit Button

In each HTML file, update the `#submitLink` and `#footerSubmit` hrefs to your Google Form URL. Or update `CONFIG.formUrl` in `js/main.js` and add JS to set the links dynamically.

## How the Approval Flow Works

1. **Someone submits** the Google Form
2. **Apps Script** sets status to "Pending" and sends an email to admins
3. **Admin receives email** with submission details and **Approve / Reject buttons**
4. **Admin clicks a button** → browser opens the Apps Script web app
5. **Apps Script processes** the action:
   - **Approve**: updates sheet, creates Google Calendar event (if applicable), triggers GitHub Actions
   - **Reject**: updates sheet, no further action
6. **GitHub Actions** runs: fetches sheet as CSV, generates JSON files, commits to repo
7. **GitHub Pages** rebuilds and serves the updated site
8. **Website** loads the JSON files and renders the approved content

## Configuration Reference

All configuration is in clearly marked sections at the top of each file:

| Setting | File | Description |
|---------|------|-------------|
| Admin emails | `Code.gs` | Who receives approval emails |
| Calendar IDs | `Code.gs` | Google Calendar per category |
| Calendar colors | `Code.gs` | Color coding for events |
| GitHub repo | `Code.gs` | Repository for webhook |
| Web app URL | `approval.gs` | Deployed Apps Script URL |
| Approval secret | `approval.gs` | Token for email link security |
| Sheet CSV URL | `build.yml` | Published sheet data URL |
| Column indices | `Code.gs` + `build.yml` | Map form fields to columns |
| Form URL | `main.js` | Google Form submission link |

## Local Development

The site is plain HTML/CSS/JS — no build tools needed.

1. Clone the repo
2. Serve locally with any static server:
   ```bash
   # Python
   python3 -m http.server 8000

   # Node
   npx serve .
   ```
3. Open `http://localhost:8000`
4. Sample data in `data/` renders immediately

## Customization

- **Colors**: Edit CSS variables in `css/style.css` (`:root` section)
- **Categories**: Update dropdowns in the Google Form, `CONFIG.CALENDAR_IDS` in Code.gs, and filter buttons in `events.html`
- **Content types**: The system supports 4 types (Event, Recommendation, Provider, Wellness Resource). Adding more requires updating the form, Apps Script column mapping, and adding a new page
